# Giscus Comments Design

## Goal

Add a serverless comment system to Jason Xu's Blog at `https://jyxu0621.github.io/blog/` using Giscus and the existing public repository `jyxu0621/blog`.

## Architecture

- GitHub Pages continues to host the static Hexo + Stellar blog.
- Giscus provides the browser-side comment widget.
- GitHub Discussions in `jyxu0621/blog` stores all comment threads and reactions.
- The Giscus GitHub App is installed only for `jyxu0621/blog`.
- Stellar's built-in Giscus adapter renders the widget on post pages; no custom comment component or separate backend is introduced.

## GitHub Configuration

1. Enable Discussions for `jyxu0621/blog`.
2. Install the Giscus GitHub App with repository access limited to `jyxu0621/blog`.
3. Use the `Announcements` discussion category. This category type restricts creation of top-level discussions to maintainers and the Giscus bot.
4. Retrieve and record the repository ID and discussion category ID generated by Giscus configuration.

## Stellar Configuration

Configure `_config.stellar.yml` with:

- `comments.service: giscus`
- Chinese UI (`data-lang: zh-CN`)
- Page-to-discussion mapping by `pathname`
- Strict mapping enabled
- Reactions enabled
- Metadata emission disabled
- Comment input above the comment list
- Theme following the visitor's preferred color scheme
- Lazy loading enabled
- Repository, repository ID, category, and category ID for `jyxu0621/blog`

Comments appear on article pages only. The home page, archives, categories, and tags pages do not receive comment widgets.

## Data Flow

1. A visitor opens an article page.
2. Stellar loads the Giscus client near the comments section.
3. Giscus maps the article's stable pathname to a Discussion in the `Announcements` category.
4. On the first comment or reaction, the Giscus bot creates the matching Discussion if one does not already exist.
5. Visitors authenticate through GitHub OAuth to comment; moderation occurs in GitHub Discussions.

## Security and Privacy

- No GitHub token, OAuth secret, or private credential is stored in the blog repository.
- Giscus App access is limited to the single public blog repository.
- Visitors must explicitly authenticate with GitHub before posting.
- Giscus is loaded only on article pages and does not add a separate analytics or advertising service.

## Failure Handling and Rollback

- If the Giscus App is not installed, Discussions are disabled, or IDs are invalid, the blog remains readable but the comment widget will fail to load.
- Deployment verification must catch missing Giscus configuration fields before publication.
- Rollback consists of removing the Giscus block from `_config.stellar.yml` and disabling the comment service; existing Discussions remain intact unless deliberately deleted later.

## Verification

1. Confirm `jyxu0621/blog` is public and Discussions are enabled.
2. Confirm the Giscus App is installed only for `jyxu0621/blog`.
3. Confirm the configured repository ID and category ID match GitHub.
4. Run source verification, clean build, production build, and generated-site verification.
5. Inspect generated post HTML for the expected Giscus parameters and ensure non-post pages do not load the widget.
6. Publish through the existing one-click workflow and confirm GitHub Actions succeeds.
7. Verify the live article page loads the Chinese Giscus widget in desktop and mobile layouts.
8. With action-time user confirmation, add one temporary test comment, verify that the matching Discussion is created, and then remove the test comment through GitHub moderation.

## Constraints

- The public blog URL remains `https://jyxu0621.github.io/blog/`.
- No separate comment repository, self-hosted service, database, personal domain, or Cloudflare deployment is introduced.
- The user must personally approve the Giscus GitHub App installation when GitHub presents the authorization page.
